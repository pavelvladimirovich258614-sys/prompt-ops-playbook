## 🎯 Введение

**Skills System** — это подход к расширению возможностей AI-агентов через модульные, переиспользуемые компоненты — **skills** (навыки). Каждый skill представляет собой инструкцию для выполнения конкретной задачи или набора задач в определённой области.

### Преимущества подхода

- 🧩 **Модульность** — навыки можно комбинировать и переиспользовать
- 📦 **Портативность** — skills работают в разных проектах
- 🔄 **Версионирование** — легко обновлять и отслеживать изменения
- 🤝 **Совместимость** — интеграция с различными AI-инструментами
- 🚀 **Масштабируемость** — от простых команд до сложных workflows

---

## 📦 Что такое Skill?

**Skill** — это структурированный файл (обычно `.md` или `.yml`), содержащий:

| Компонент | Описание |
|-----------|----------|
| **Назначение** | Что делает этот skill |
| **Триггеры** | Когда активировать skill |
| **Инструкции** | Пошаговые действия для выполнения |
| **Параметры** | Входные данные и конфигурация |
| **Примеры** | Образцы использования |

```markdown
|----------|-----|----------|

## Инструкции
1. Шаг первый
2. Шаг второй
3. Шаг третий

## Примеры
### Пример 1
Вход: ...
Выход: ...
```

## 🚀 Как использовать Skills

### Базовое использование

```bash
# Активация skill через Claude Code
@skill название-скилла

# С параметрами
@skill название-скилла param1=value1 param2=value2
```

### Интеграция с Subagent-Driven Development

```
┌─────────────────────────────────────────────────────────┐
│                    🎛️ Orchestrator                       │
│                    (загружает skills)                    │
└─────────────────────┬───────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        │             │             │
        ▼             ▼             ▼
┌───────────┐  ┌───────────┐  ┌───────────┐
│ 🛠️ Skill  │  │ 🛠️ Skill  │  │ 🛠️ Skill  │
│   Code    │  │   Test    │  │   Docs    │
└───────────┘  └───────────┘  └───────────┘
```

### Пример workflow с skills

1. **@skill analyze-requirements** — анализ требований
2. **@skill generate-code** — генерация кода (параллельно)
3. **@skill write-tests** — написание тестов (параллельно)
4. **@skill review-code** — код-ревью
5. **@skill deploy** — деплой

## 🔨 Создание собственных Skills

### Шаблон skill-файла

```markdown
# Skill: my-custom-skill

## Описание
Описание того, что делает этот skill

## Триггеры
- trigger-word-1
- trigger-word-2

## Параметры
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| input | string | Да | Входные данные |
| output | string | Нет | Путь к выходному файлу |

## Инструкции
1. Проанализировать входные данные
2. Выполнить основную логику
3. Вернуть результат

## Примеры
### Пример 1: Базовое использование
```
Вход: данные
Результат: обработанные данные
```

## Примечания
- Важное примечание 1
- Важное примечание 2
```

### Рекомендации по созданию

✅ **Делайте:**
- Используйте чёткие, описательные названия
- Документируйте все параметры
- Приводите примеры использования
- Тестируйте skill изолированно
- Версионируйте изменения

❌ **Не делайте:**
- Не создавайте слишком большие skills
- Избегайте неявных зависимостей
- Не забывайте про обработку ошибок
