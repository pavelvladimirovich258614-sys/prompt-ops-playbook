# Skill: Brainstorming

## Описание
Socratic refinement of ideas through targeted questioning before writing code.

## Тип
Technique

## Когда использовать
- Перед началом новой задачи
- При проектировании архитектуры
- При выборе технологий
- При определении scope работ

## Предварительные условия
- Понимание бизнес-проблемы
- Контекст проекта
- Доступ к требованиям

## Пошаговая инструкция

### 1. Описать проблему
Чётко сформулировать, какую проблему решаем.

```
Проблема: [Описание]
Пользователь: [Кто столкнулся]
Текущее поведение: [Что происходит]
Ожидаемое поведение: [Что должно происходить]
```

### 2. Задать уточняющие вопросы
- Какие есть ограничения?
- Какие edge cases возможны?
- Как это масштабируется?
- Какие есть альтернативы?

### 3. Исследовать граничные случаи
- Что при пустом input?
- Что при очень большом input?
- Что при ошибках?
- Что при конкурентном доступе?

### 4. Уточнить требования
- Functional requirements
- Non-functional requirements
- Constraints
- Acceptance criteria

### 5. Получить approval на дизайн
**HARD-GATE:** Ни строчки кода до approval.

```
Дизайн: [Описание решения]
Trade-offs: [Что учли]
Решение: [Почему так]
Approval: [Да/Нет]
```

## Антипаттерны

### ❌ НЕ делать
- Начинать писать код до approval
- Пропускать этап исследования
- Игнорировать edge cases
- Не документировать trade-offs

### ⚠️ Распространённые ошибки
- Поспешное кодирование
- Недостаточное понимание проблемы
- Пропуск нефункциональных требований

## Примеры

### Пример 1: API Endpoint

**Проблема:** Нужен endpoint для получения списка пользователей

**Вопросы:**
- Пагинация? Да, 50 записей по умолчанию
- Сортировка? По created_at desc
- Фильтры? По role, status
- Авторизация? Только admin

**Edge Cases:**
- Пустой список → 200 с пустым массивом
- Невалидный page → 400
- Нет прав → 403

**Дизайн:**
```
GET /api/users?page=1&limit=50&role=admin
Response: { users: [], total: 0, page: 1 }
```

### Пример 2: Background Job

**Проблема:** Нужно обрабатывать загрузку файлов асинхронно

**Вопросы:**
- Максимальный размер файла? 100MB
- Форматы? PDF, DOC, DOCX
- Очередь? Redis + Sidekiq
- Retry policy? 3 попытки с backoff

**Edge Cases:**
- Файл больше лимита → ошибка
- Неподдерживаемый формат → ошибка
- Ошибка обработки → retry
