# Concept: Workflow

## Что такое Workflow

Workflow — это обязательный агентский процесс, который автоматически активируется на основе контекста. В отличие от skills, workflows выполняются всегда и не требуют явного вызова.

## Core Workflows

### 1. Brainstorming
**Когда:** Перед написанием любого кода

**Что делает:**
- Задаёт уточняющие вопросы
- Исследует граничные случаи
- Уточняет требования
- Получает approval на дизайн

**HARD-GATE:** Ни строчки кода до approval дизайна.

### 2. Using Git Worktrees
**Когда:** При работе с новой веткой

**Что делает:**
- Создаёт изолированное рабочее пространство
- Позволяет переключаться между задачами
- Сохраняет контекст каждой ветки

```bash
# Создать worktree
git worktree add -b feature-branch ../feature-branch

# Перейти в worktree
cd ../feature-branch
```

### 3. Writing Plans
**Когда:** Перед началом реализации

**Что делает:**
- Разбивает работу на задачи по 2-5 минут
- Создаёт чек-лист выполнения
- Определяет порядок действий

**Формат плана:**
```markdown
## Plan: Название задачи

### Задачи
- [ ] Задача 1 (2 мин)
- [ ] Задача 2 (5 мин)
- [ ] Задача 3 (3 мин)

### Зависимости
- Зависимость 1
- Зависимость 2

### Критерии готовности
- [ ] Критерий 1
- [ ] Критерий 2
```

### 4. Subagent-driven Development
**Когда:** Для быстрой итерации

**Что делает:**
- Создаёт свежего субагента на каждую задачу
- Выполняет двухэтапное ревью
- Гарантирует чистый контекст

**Процесс:**
1. Создать субагента для задачи
2. Делегировать выполнение
3. Двухэтапное ревью:
   - Spec Compliance Review
   - Code Quality Review
4. Интеграция изменений

### 5. Test-Driven Development
**Когда:** Всегда при написании production кода

**Железный закон:**
> NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST

Если код написан до теста — он удаляется.

**Цикл:**
1. **RED** — Написать тест (должен падать)
2. **GREEN** — Написать минимальный код (тест проходит)
3. **REFACTOR** — Улучшить код (тесты проходят)

### 6. Requesting Code Review
**Когда:** Перед созданием PR

**Что делает:**
- Проверяет соответствие плану
- Сообщает о проблемах по severity
- Готовит summary изменений

### 7. Finishing a Development Branch
**Когда:** Завершение работы над веткой

**Что делает:**
- Проверяет тесты
- Предлагает варианты merge
- Готовит PR

## Workflow vs Skill

| Workflow | Skill |
|----------|-------|
| Обязателен к выполнению | Применяется по необходимости |
| Автоматическая активация | Явный выбор агента |
| Контролирует процесс | Предоставляет инструкции |
| HARD-GATE | Рекомендация |
