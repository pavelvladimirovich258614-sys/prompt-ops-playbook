# Concept: Skill

## Что такое Skill

Skill — это структурированный справочник по проверенной технике, паттерну или инструменту, который помогает агентам находить и применять эффективные подходы.

## Типы Skills

### Technique
Конкретный метод с пошаговыми инструкциями.

**Пример:** Systematic Debugging — 4-фазный процесс расследования проблем.

### Pattern
Способ мышления о проблемах, подход к решению.

**Пример:** Subagent-driven Development — делегирование задач свежим субагентам.

### Reference
API документация, справочник по синтаксису, документация инструмента.

**Пример:** Документация по Git worktrees.

## Структура Skill

```
skills/
└── <skill-name>/
    ├── SKILL.md          # Обязательно. Основная документация
    ├── config.json       # Опционально. Конфигурация
    ├── examples/         # Опционально. Примеры использования
    └── templates/        # Опционально. Шаблоны
```

## Формат SKILL.md

```markdown
# Skill Name

## Описание
Краткое описание назначения skill.

## Когда использовать
- Ситуация 1
- Ситуация 2

## Предварительные условия
Что нужно знать/иметь перед применением.

## Пошаговая инструкция
1. Шаг 1
2. Шаг 2
3. Шаг 3

## Примеры
### Пример 1: Название
```code
// пример кода
```

## Антипаттерны
- Что делать нельзя
- Распространённые ошибки

## Связанные skills
- [Skill 1](../other-skill/)
- [Skill 2](../another-skill/)

## Источники
- Ссылка на источник
```

## Пример: Brainstorming Skill

```markdown
# Brainstorming

## Описание
Socratic refinement of ideas through targeted questioning before writing code.

## Когда использовать
- Перед началом новой задачи
- При проектировании архитектуры
- При выборе технологий

## Предварительные условия
- Понимание проблемы
- Контекст проекта

## Пошаговая инструкция
1. Описать проблему
2. Задать уточняющие вопросы
3. Исследовать граничные случаи
4. Уточнить требования
5. Получить approval на дизайн

## Антипаттерны
- НЕ начинать писать код до approval
- НЕ пропускать этап исследования
```
