# Prompt Ops Playbook

## Что такое Prompt Ops

Prompt Ops — это систематический подход к проектированию, развёртыванию и управлению промптами для AI-агентов.

## Core Principles

### 1. Clarity (Ясность)
Промпт должен однозначно передавать намерение.

```
❌ Плохо: "Сделай что-нибудь с этим"
✅ Хорошо: "Рефактори функцию processData для улучшения читаемости"
```

### 2. Context (Контекст)
Предоставлять необходимый контекст без перегрузки.

```
❌ Плохо: "Исправь баг" (нет контекста)
✅ Хорошо: "Исправь баг в функции calculateTotal: при отрицательном input возвращает NaN вместо 0"
```

### 3. Control (Контроль)
Чёткие границы и критерии успеха.

```
❌ Плохо: "Улучши код"
✅ Хорошо: "Улучши производительность функции на 20%, сохранив поведение. Измерь до и после."
```

## The Prompt Ops Cycle

```
┌─────────────┐
│   DEFINE    │ → Что нужно достичь?
└──────┬──────┘
       ▼
┌─────────────┐
│   DESIGN    │ → Как структурировать промпт?
└──────┬──────┘
       ▼
┌─────────────┐
│   DEPLOY    │ → Где и как использовать?
└──────┬──────┘
       ▼
┌─────────────┐
│   EVAL      │ → Насколько хорошо работает?
└──────┬──────┘
       ▼
┌─────────────┐
│   ITERATE   │ → Как улучшить?
└─────────────┘
```

## Prompt Patterns

### Zero-shot
```
Переведи на Python:
def hello_world():
    print("Hello, World!")
```

### Few-shot
```
Примеры:
Input: "hello" → Output: "HELLO"
Input: "world" → Output: "WORLD"

Input: "test" → Output:
```

### Chain of Thought (CoT)
```
Реши задачу пошагово:
1. Определи переменные
2. Найди формулу
3. Подставь значения
4. Вычисли результат

Задача: [описание]
```

### ReAct (Reasoning + Acting)
```
Думай шаг за шагом и используй инструменты:
Thought: Мне нужно найти информацию о...
Action: search("query")
Observation: Результаты поиска...
Thought: Теперь я могу...
```

## Context Management

### Структура контекста
```
1. System Prompt (роль, правила)
2. User Prompt (запрос)
3. History (история)
4. Tools (доступные инструменты)
5. Examples (примеры)
```

### Техники сжатия
- Summarization: краткое содержание длинных текстов
- Selection: выбор релевантных частей
- Compression: удаление несущественного

## Tool Integration (MCP)

### Model Context Protocol
Стандарт для подключения инструментов к AI-агентам.

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<token>"
      }
    }
  }
}
```

### Типы инструментов
- **File System**: чтение/запись файлов
- **Search**: поиск по коду
- **Execute**: выполнение команд
- **API**: внешние сервисы

## Evaluation Framework

### Метрики
- **Accuracy**: правильность результата
- **Completeness**: полнота выполнения
- **Efficiency**: количество шагов/токенов
- **Safety**: отсутствие вредоносных действий

### Методы оценки
1. **Human Review**: экспертная оценка
2. **Automated Tests**: автоматические тесты
3. **A/B Testing**: сравнение вариантов
4. **User Feedback**: обратная связь пользователей

## Best Practices

### DO (Делать)
- ✅ Быть конкретным
- ✅ Предоставлять примеры
- ✅ Определять формат вывода
- ✅ Указывать ограничения
- ✅ Тестировать промпты

### DON'T (Не делать)
- ❌ Быть расплывчатым
- ❌ Игнорировать edge cases
- ❌ Перегружать контекст
- ❌ Забывать о безопасности
- ❌ Не проверять результаты

- [ ] Указаны ограничения
- [ ] Предоставлены примеры
- [ ] Проверен на edge cases
