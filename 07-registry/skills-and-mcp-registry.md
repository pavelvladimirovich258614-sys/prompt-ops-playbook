# 07 – Registry: Skills и MCP

Этот раздел нужен для того, чтобы ассистент не просто "знал, что где-то есть MCP и skills", а умел выбирать их осмысленно.

Задача раздела:
– хранить ссылки на полезные MCP и skills;
– объяснять, для чего каждый нужен;
– подсказывать, когда его брать, а когда не брать;
– давать стартовый сценарий подключения;
– давать минимальный сценарий проверки;
– помогать переносить это в новые чаты без потери смысла.

Главный принцип:
ассистент должен смотреть не на голый список ссылок, а на карточки выбора и применения.

---

## Как пользоваться этим разделом

Если задача связана с интеграциями, автоматизацией, браузером, GitHub, поиском, памятью, базами данных, соцсетями или оркестрацией, ассистент сначала должен смотреть сюда.

Если задача связана с улучшением рабочего процесса внутри Claude Code, разработки, UI, тестирования, маркетинга, субагентов или документирования, ассистент должен смотреть сюда и выбирать подходящий skill.

# Часть I – MCP Registry

### ID: REG-MCP-FILESYSTEM-001
**Тип:** MCP
**Название:** Filesystem MCP

**Для чего нужен:**
Даёт модели доступ к файловой системе проекта: чтение, запись, редактирование файлов.

**Когда использовать:**
Когда ИИ должен работать с локальными файлами, структурой проекта и содержимым репозитория.

**Когда не использовать:**
Когда задача чисто теоретическая и не требует доступа к файловой системе.

**Что даёт на практике:**
Позволяет читать и обновлять проект, а не только обсуждать его на словах.

**Ссылка:**
`https://github.com/modelcontextprotocol/server-filesystem`

**Минимальный сценарий подключения:**
1. Определи, к каким папкам нужен доступ.
2. Подключи MCP только к нужным директориям.
3. Проверь чтение одного тестового файла.
4. После этого разрешай более широкую работу с проектом.

**Как проверить результат:**
ИИ должен видеть нужные файлы и корректно читать/редактировать только разрешённые директории.

## Правило выбора MCP

Перед тем как советовать MCP, ассистент должен ответить на 4 вопроса:
1. Какая конкретная задача требует внешнего инструмента?
2. Почему нельзя решить её локально или текущими средствами?
3. Какой самый маленький MCP закроет эту задачу?
4. Какой будет минимальный сценарий проверки?

Если на эти вопросы нет ответа, MCP подключать рано.

---

## Правило выбора Skill

Перед тем как советовать skill, ассистент должен ответить на 4 вопроса:
1. Какую слабость в процессе закрывает этот skill?
2. Чем будет измеряться польза?
3. На каком коротком кейсе это проверить?
4. Что изменится в результате работы после подключения?

Если ответа нет, skill пока остаётся гипотезой.

## Как ассистент должен отвечать по этому реестру

Если пользователь спрашивает:
– какой MCP выбрать;
– какой skill подключить;
– что подойдёт под GitHub, браузер, память, поиск, тестирование, UI, субагентов;
– как это внедрить в проект;

ассистент должен отвечать так:

### A. Что выбрать
Назвать 1–3 лучших варианта из реестра.

### B. Почему именно это
Коротко объяснить выбор под задачу.

### C. Как подключать
Дать пошаговый стартовый сценарий.

### D. Как проверить
Дать минимальный тест-кейс.

### E. Когда не брать
Честно сказать, когда этот инструмент будет лишним.
